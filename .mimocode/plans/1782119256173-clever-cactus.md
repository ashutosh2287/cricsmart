# CricSmart Deployment Plan — Vercel

## Overview
Deploy CricSmart to Vercel with Neon (PostgreSQL) and Upstash (Redis). Free tier for all services.

---

## Step 1: Set Up PostgreSQL Database (Neon)

1. Go to https://neon.tech and sign up (free)
2. Create a new project:
   - Project name: `cricsmart`
   - Region: closest to your users
3. Copy the connection string from the dashboard:
   ```
   postgresql://username:password@ep-xxx.us-east-2.aws.neon.tech/cricsmart?sslmode=require
   ```
4. Save this as `DATABASE_URL` — you'll need it later

---

## Step 2: Set Up Redis (Upstash)

1. Go to https://upstash.com and sign up (free)
2. Create a new Redis database:
   - Name: `cricsmart`
   - Region: closest to your Vercel region
3. Copy the `REDIS_URL` from the Connect tab
4. Save this — you'll need it later

---

## Step 3: Get CricAPI Key

1. Go to https://cricapi.com and sign up
2. Get your API key from the dashboard
3. Save as `CRICKET_API_KEY`

---

## Step 4: Push Code to GitHub

```bash
cd "C:\Final year project\cricket-analytics"
git add -A
git commit -m "Major UI upgrade: animations, redesigned pages, new components"
git push origin main
```

---

## Step 5: Deploy to Vercel

1. Go to https://vercel.com and sign in with GitHub
2. Click "Add New Project"
3. Import your GitHub repository (`cricsmart`)
4. Configure:
   - Framework Preset: Next.js
   - Build Command: `npm run build`
   - Install Command: `npm ci`
5. **DO NOT** click Deploy yet — add environment variables first

---

## Step 6: Configure Environment Variables in Vercel

Add these in the Vercel dashboard under Settings → Environment Variables:

### Required:
| Variable | Value |
|----------|-------|
| `DATABASE_URL` | Your Neon connection string |
| `REDIS_URL` | Your Upstash Redis URL |
| `CRICKET_API_KEY` | Your CricAPI key |

### Auth (use these values):
| Variable | Value |
|----------|-------|
| `AUTH_ENABLED` | `true` |
| `AUTH_ROLLOUT_PHASE` | `1` |
| `AUTH_COOKIE_NAME` | `cricsmart_session` |
| `AUTH_SESSION_TTL_SECONDS` | `604800` |
| `AUTH_SESSION_ROTATE_SECONDS` | `1800` |
| `AUTH_RATE_LIMIT_WINDOW_SECONDS` | `60` |
| `AUTH_RATE_LIMIT_MAX_ATTEMPTS` | `6` |
| `AUTH_ALLOW_DEV_BYPASS` | `false` |
| `AUTH_ENFORCE_SSE` | `true` |

### App Config:
| Variable | Value |
|----------|-------|
| `NEXT_PUBLIC_BASE_URL` | `https://your-project.vercel.app` |
| `NEXT_PUBLIC_ERROR_ENDPOINT` | (leave empty) |
| `LOG_LEVEL` | `info` |
| `CRICAPI_POLL_INTERVAL_MS` | `60000` |
| `SOCKET_ALLOWED_ORIGINS` | `https://your-project.vercel.app` |

## Step 7: Run Prisma Migration

Before first deploy, the database needs tables. Two options:

### Option A: Vercel Build (automatic)
Add to `package.json` scripts:
```json
"build": "prisma generate && prisma db push && next build"
```

### Option B: Local push (recommended)
```bash
cd "C:\Final year project\cricket-analytics"
npx prisma db push
```

This creates all tables in your Neon database.

## Troubleshooting

### Build fails with Prisma error
- Ensure `DATABASE_URL` is set in Vercel env vars
- The build script runs `prisma generate` automatically via `postinstall`

### Redis connection errors
- Redis is optional — app works without it
- Sessions and rate limiting won't work without Redis
- Upstash Redis URL must start with `rediss://` for TLS

### Auth not working
- Ensure `AUTH_ENABLED=true`
- Clear browser cookies and try again
- Check that `SOCKET_ALLOWED_ORIGINS` matches your Vercel URL

### Live fixtures not loading
- Ensure `CRICKET_API_KEY` is set
- Check https://cricapi.com dashboard for API quota
