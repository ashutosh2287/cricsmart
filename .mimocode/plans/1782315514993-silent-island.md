# Plan: UI Animations, Layout Fixes & Simulation Bug Fix

## Issues Identified

### 1. Right rail empty on all tabs
**Root cause**: `src/app/match/[runtimeMatchId]/page.tsx:1213` — inline `style={{ display: "none" }}` overrides the `lg:block` Tailwind class, making `StickyInsightsRail` permanently hidden.

### 2. Simulation stuck
**Root cause**: `src/services/simulation/matchSimulator.ts` — the `runBall` function has multiple early-return paths (replay mode check at ~line 480, paused check at ~line 493, etc.) that do NOT schedule the next `setTimeout(runBall, control.speed)`. Once any early return fires, the loop dies permanently.

### 3. Background animation not continuous
The layout (`src/app/layout.tsx:56-60`) has static glowing orbs. They need CSS keyframe animations for continuous movement.

### 4. Admin tab has unused components
Admin tab includes `AdminScoringPanel`, `BroadcastDirectorPanel`, `BroadcastControlDashboard` — user wants only simulation controls kept.

### 5. Delete button on match page
Matches page (`src/app/matches/page.tsx`) already has delete. Need to add one to the match detail page admin tab.

### Step 2: Fix simulation stuck bug
**File**: `src/services/simulation/matchSimulator.ts`
- In the `runBall` function (~line 473-843), every early `return` path must schedule `control.timeoutRef = setTimeout(runBall, control.speed)` before returning, OR the `finally` block must handle re-scheduling
