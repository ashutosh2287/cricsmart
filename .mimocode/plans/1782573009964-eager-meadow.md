# Landing Page Animation & Interaction Overhaul

## Problem Summary

1. **Double footer** — Root layout and LandingPageClient both render `<AppFooter />`
2. **Flat landing page** — Minimal hover effects, no scroll-triggered reveals on most sections
3. **Logo has no hover animation** — Static text, no interactive feedback
4. **Profile avatar has no hover animation** — Plain circle in navbar
5. **No avatar file upload** — Users must paste external URLs

---

## Implementation Plan

### Fix 1: Double Footer (5 min)

**Root cause:** `src/app/layout.tsx` renders `<AppFooter />` on ALL pages. `LandingPageClient` also renders its own `<AppFooter />`.

**Fix:**
- Remove `<AppFooter />` from `src/app/layout.tsx`
- Keep `<AppFooter />` in `LandingPageClient.tsx` (landing page)
- Add `<AppFooter />` to `AppShellContent.tsx` (authenticated pages)

**Files:**
- `src/app/layout.tsx` — remove AppFooter import + render
- `src/components/layout/AppShellContent.tsx` — add AppFooter after main content

---

### Fix 2: Logo Hover Animation (20 min)

**Goal:** Unique, memorable animation when hovering the logo in navbar and landing page.

**Design:**
- On hover: logo icon scales up slightly (1.1x), gains a cyan glow shadow, and the gradient shifts
- On hover: "Cric" text stays gradient, "Lens" text gets a subtle glow
- Smooth spring transition

**Implementation:**
- Wrap logo in `motion.div` with `whileHover` variants
- Add glow shadow using `--shadow-glow-cyan`
- Add subtle rotation (2deg) for playfulness

**Files:**
- `src/components/landing/LandingPageClient.tsx` — wrap nav logo in motion.div
- `src/components/layout/Navbar.tsx` — wrap app logo in motion.div

**New component:** None needed — use inline framer-motion variants.

---

### Fix 3: Profile Avatar Hover Animation (15 min)

**Goal:** Interactive feedback when hovering the profile icon in navbar.

**Design:**
- On hover: avatar scales up (1.08x), border glows cyan, subtle rotation (3deg)
- Ring pulse effect on hover

**Implementation:**
- Wrap `UserAvatar` in `motion.div` in the Navbar
- Add `whileHover` with scale, boxShadow, and rotate

**Files:**
- `src/components/layout/Navbar.tsx` — wrap avatar link in motion.div
- `src/components/account/UserAvatar.tsx` — accept optional `className` prop for external animation

### Fix 5: Enhanced Footer with Animations (30 min)

**Goal:** Footer should feel premium, not plain.

**Design:**
- Staggered reveal of footer columns on scroll (using `whileInView`)
- Social icons have hover glow + scale effect
- Footer links have underline slide animation on hover
- Copyright text fades in last

**Implementation:**
- Wrap footer columns in `motion.div` with `stagger` container
- Each column uses `fadeUp` variant
- Social icons use `springPop` on hover
- Links get CSS `::after` underline animation

**Files:**
- `src/components/layout/AppFooter.tsx` — full rewrite with animations

---

### Fix 6: Avatar File Upload (45 min)

**Goal:** Users can upload profile pictures directly, not just paste URLs.

**Design:**
- Click avatar → file picker opens
- Selected image shows preview
- Image is converted to base64 data URL and saved
- Max file size: 2MB
- Accepted: JPG, PNG, WebP

**Implementation:**

#### 6a. AvatarUpload Component
New component at `src/components/ui/AvatarUpload.tsx`:
- Renders current avatar with edit overlay on hover
- Click opens hidden `<input type="file">`
- On file select: validate size/type, read as base64, show preview
- Calls `onUpload(dataUrl)` callback

#### 6b. Profile Page Integration
- Replace URL input with AvatarUpload component
- Keep URL input as fallback/advanced option
- Add drag-and-drop support

**Files:**
- `src/components/ui/AvatarUpload.tsx` — NEW component
- `src/app/account/profile/ProfileClient.tsx` — integrate AvatarUpload

---

### Fix 7: Animation Hierarchy (15 min)

**Goal:** Higher-importance elements get more dramatic animations.

**Hierarchy:**
| Level | Elements | Animation |
|-------|----------|-----------|
| **Hero** | Hero text, CTA buttons, product preview | `heroReveal` + `springBounce` + glow |
| **Primary** | Feature cards, stats, live matches | `magneticHover` + `fadeUp` + stagger |
| **Secondary** | Footer columns, nav links | `fadeUp` + subtle hover |
| **Tertiary** | Captions, badges, small text | `fadeIn` only |

**Implementation:** Apply variants consistently across LandingPageClient.
